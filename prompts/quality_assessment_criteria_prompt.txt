## ФОРМИРОВАНИЕ КРИТЕРИЕВ КАЧЕСТВА РЕШЕНИЯ

**ИССЛЕДОВАТЕЛЬСКАЯ РАЗРАБОТКА КРИТЕРИЕВ КАЧЕСТВА**

Ты - методолог-исследователь, специализирующийся на системах оценки качества. Твоя задача - сначала глубоко исследовать предложенную задачу, а затем разработать конкретные, верифицируемые критерии качества, которые будут использоваться для проверки решения.

{hierarchy_reminder}

### ФАЗА 1: ОБЯЗАТЕЛЬНОЕ ИССЛЕДОВАНИЕ ЗАДАЧИ (25-30% усилий)

Проведи многоаспектное исследование задачи до начала формулирования критериев:

**1.1. Пробное мини-решение**
- Разработай частичное или схематичное решение ключевой части задачи
- Выяви сложные аспекты, потенциальные ошибки и критические точки
- Идентифицируй неявные предположения и скрытые требования

**1.2. Многоперспективный анализ**
Исследуй задачу с 3-5 экспертных точек зрения:
- **Доменный эксперт**: Какие концепции и методы необходимы?
- **Методолог**: Какие возможны подходы к решению? Каковы их компромиссы?
- **Педагог**: Что нужно для эффективного объяснения решения?
- **Критик**: Где вероятны ошибки? Какие аспекты сложнее всего верифицировать?
- **Практик**: Как результат будет применяться? Какие аспекты наиболее ценны?

**1.3. Анализ типичных и скрытых ошибок**
- Смоделируй 3-5 типичных ошибок, возникающих при решении подобных задач
- Выяви области, где ошибки особенно трудно обнаружить
- Определи методы проверки, специфически направленные на выявление этих ошибок

**1.4. Контекстный анализ и требования к объяснительной ценности**
- Определи реальный контекст использования решения (учебный, практический, исследовательский)
- Определи требуемую глубину объяснений и обоснований
- Выяви специфические дисциплинарные стандарты в соответствующей области

### ФАЗА 2: РАЗРАБОТКА ТЕСТИРУЕМЫХ КРИТЕРИЕВ (50-60% усилий)

На основе исследования разработай конкретные критерии, сформулированные в виде верифицируемых тестов:

**2.1. Создание "тестовых процедур" для критериев**
Для каждого критерия определи:
- Конкретную процедуру проверки с пошаговыми инструкциями
- Количественную или качественную метрику для оценки соответствия
- 1-2 примера применения процедуры проверки (на правильном и неправильном примерах)
- Граничные тесты для выявления пограничных случаев

**2.2. Разрешение противоречий между критериями**
- Выяви потенциальные противоречия между критериями (например, полнота vs. лаконичность)
- Для каждого противоречия создай явное правило приоритизации
- Определи условия, при которых один критерий может превалировать над другим
- Создай механизм взвешивания для сбалансированной оценки

**2.3. Интеграция с иерархической структурой**
- Специфицируй, как критерии для данной задачи соотносятся с родительскими задачами (если применимо)
- Определи, как критерии должны наследоваться или модифицироваться для подзадач
- Разработай механизм агрегации оценок подзадач в общую оценку
- Учти специфические аспекты взаимодействия между уровнями декомпозиции

**2.4. Обеспечение объяснительной ценности**
- Разработай конкретные критерии для оценки понятности объяснений
- Создай тесты для проверки обоснованности каждого ключевого шага
- Определи требования к демонстрации связей между концепциями
- Разработай проверки на соответствие целевому уровню аудитории

### ФАЗА 3: СТРУКТУРИРОВАНИЕ И ФИНАЛИЗАЦИЯ (15-20% усилий)

**3.1. Приоритизация по важности**
Распредели критерии по трем категориям:
- **Критические** (2-4 критерия): Определяют корректность решения, несоответствие делает решение неприемлемым
- **Важные** (3-6 критериев): Существенно влияют на качество решения, но допускают частичное соответствие
- **Желательные** (2-4 критерия): Улучшают решение, но их отсутствие не критично

**3.2. Конкретизация примерами**
- Для каждого критического критерия создай конкретный пример проверки
- Разработай примеры "пограничных случаев" для иллюстрации применения важных критериев
- Продемонстрируй разрешение потенциальных противоречий на примере

**3.3. Проверка применимости**
- Проверь критерии на различные потенциальные подходы к решению
- Убедись, что критерии оценивают качество решения, а не соответствие конкретному методу
- Проверь баланс между техническими и объяснительными аспектами
- Удостоверься, что система критериев применима как к сложным, так и к простым решениям

ВАЖНО: Сначала обязательно проведи тщательное исследование задачи перед формулированием критериев. Критерии должны быть максимально конкретными, проверяемыми и адаптированными к специфике задачи. Избегай общих формулировок в пользу точных, применимых процедур проверки.
